# 霸业：三国崛起 接手执行计划

## 目标

在不破坏当前可玩原型的前提下，把项目从“单文件可运行原型”推进到“可持续迭代的策略游戏基础版本”。

本计划同时覆盖产品体验、数值平衡、内容建设与工程落地顺序。

## 当前判断

- 当前主循环已经成立：月度经营、探访招贤、征兵训练、攻城扩张。
- 主要问题不在于“没有系统”，而在于“系统之间还没有形成稳定闭环”。
- 最明显的风险来自三类：单城结算、数值膨胀、内容反馈不足。

## 里程碑

### M1 基础数值与规则收敛

目标：先把游戏变成一个可持续调数值、可稳定体验前 12 回合的版本。

交付项：

- 抽离初始配置、关键公式、平衡参数。
- 修正经济曲线，避免开局长期失血与中期爆炸并存。
- 把武将能力从简单总和改为“主将 + 辅佐折算”。
- 收敛探访、征兵、训练、战斗的收益波动。
- 修正关键胜利结算与日志反馈问题。

验收标准：

- 前 6 回合内，玩家不乱征兵时可维持基本经营。
- 玩家可通过内政或探访建立不同节奏，而不是被单一路线绑死。
- 新招募武将会明显增强势力，但不会立刻导致公式失控。

### M2 系统闭环补齐

目标：让忠诚、外交、城市经营不再只是显示项，而是实际决策系统。

交付项：

- 忠诚度后果：拒战、降效、叛逃、被挖角。
- 外交后果：停战、通商、施压、求援、劝降。
- 多城月结算：收入、驻军、民心、城池分工。
- 城池差异化：资源型、军事型、人才型城市。

验收标准：

- 玩家在中盘会面临“扩张、稳内政、做外交”的真实取舍。
- 忠诚与外交不再是装饰系统。

### M3 内容扩展与重复可玩性

目标：把现有骨架扩展为具有中长期游玩动力的三国策略体验。

交付项：

- 城市事件池与地区差异内容。
- 武将特技、人物关系、相性与定位。
- 宝物装备化与争夺事件。
- 战报层、兵种层、战法层逐步拓展。
- AI 主动行为与阶段目标。

验收标准：

- 玩家每局的开局路线、招募顺序和扩张路径出现差异。
- 武将、城市和外交对象具备明确辨识度。

## 当前执行顺序

1. 先做 M1，避免在失控公式上堆新内容。
2. M1 内部优先级如下：
   1. 配置与公式外置。
   2. 经济与战斗重平衡。
   3. 胜负反馈与日志修正。
3. M2 在 M1 稳定后推进，先补后果，再补复杂表现。
4. M3 最后进入，防止内容建设被底层频繁返工吞掉。

## 本次已启动项

- 新增执行计划文档。
- 启动 M1 第 1 项：关键数值配置与公式抽离。
- 启动 M1 第 2 项的首轮实现：经济、征兵、训练、战斗、探访收益重平衡。
- 启动 M1 第 3 项：把单城逻辑推进到多城基础版本，先完成多城月结算与当前操作城切换。
- 启动 M2 第 1 项：让忠诚度进入能力计算与月度事件，形成基础闭环。

## 后续建议协作方式

- 每完成一个可验证切片，就同步更新本计划。
- 数值变更优先保留为配置项，不继续回写死到单文件组件里。
- 当 M1 基本稳定后，再进入多城与外交闭环开发。
